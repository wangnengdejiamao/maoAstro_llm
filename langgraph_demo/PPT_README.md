## PPT 结构概览

### 封面 (1页)
- 标题：智能天文数据分析系统
- 副标题：基于 LangGraph 的多智能体架构与知识蒸馏技术

### 第一章：背景与动机 (3页)
1. **天文数据的挑战**
   - 数据爆炸：LAMOST 2000万+光谱
   - 数据多样性：光谱、测光、时序、文献
   - 专业知识门槛
   - 传统方法局限

2. **AI 能带来什么？**
   - 自然语言交互
   - 跨数据库整合
   - 科学推理能力
   - 持续学习

### 第二章：核心概念解析 (6页)
1. **什么是大语言模型？**
   - 类比：超级图书馆员
   - 核心能力：理解 + 生成 + 推理
   - 参数规模：Qwen-7B = 70亿参数

2. **RAG：检索增强生成**
   - 左栏：传统大模型的问题（知识截止、幻觉）
   - 右栏：RAG 解决方案（外挂知识库、查询数据库）

3. **什么是 Tool（工具）？**
   - 类比：科学家的仪器
   - Query_VSX、Query_LAMOST、Spectral_Analysis

4. **什么是 Agent（智能体）？**
   - 类比：研究助理
   - 感知 → 规划 → 执行 → 记忆

5. **什么是 LangGraph？**
   - 类比：科研项目流程图
   - 节点、边、状态、循环

### 第三章：系统架构设计 (3页)
1. **系统整体架构图**
   - 展示完整的工作流图 (langgraph_workflow.png)

2. **六大智能体分工**
   - Router Agent、Data Retrieval Agent
   - Spectral Analysis Agent、Reasoning Agent (Qwen)
   - Verification Agent、Output Agent

### 第四章：知识蒸馏技术 (4页)
1. **大模型 vs 小模型对比**
   - 左栏：Qwen-72B（参数720亿，显存144GB）
   - 右栏：Qwen-7B（参数70亿，显存16GB）

2. **知识蒸馏架构图**
   - 展示蒸馏架构 (distillation_architecture.png)

3. **蒸馏的三重损失**
   - 硬损失（Hard Loss）
   - 软损失（Soft Loss）
   - 任务损失（Task Loss）

4. **性能对比结果**
   - 展示性能对比图 (performance_comparison.png)

### 第五章：应用与展望 (3页)
1. **实际应用场景**
   - 变星识别、光谱分析、文献综述、教学辅助

2. **项目创新点**
   - 首创性、实用性、高效性、可扩展性

3. **未来展望**
   - 多模态融合、实时学习、协同观测、全球协作

### 总结与致谢 (2页)
1. **核心要点回顾**
   - 五大要点总结

2. **致谢页**
   - 感谢聆听 + 项目链接

## 使用建议

### 演讲时长
- **总时长**：建议 30-45 分钟
- **每页讲解**：1-2 分钟
- **重点页面**（概念解释）：可适当延长

### 适合听众
- 天文学研究者
- 计算机/AI 研究者
- 对交叉学科感兴趣的学生
- 科研机构管理人员

### 演讲技巧
1. **第二章（概念解释）**
   - 多使用类比（图书馆员、仪器、助理）
   - 强调"原来AI是这样工作的"

2. **第三章（系统架构）**
   - 指着图讲解数据流向
   - 强调每个Agent的作用

3. **第四章（知识蒸馏）**
   - 对比左右栏，突出"大模型变小模型"的效果
   - 解释三重损失时可用公式辅助

## 修改指南

### 修改标题
```python
# 编辑 create_presentation.py 中的
pres.add_title_slide("你的标题", "你的副标题", "报告人名字")
```

### 修改配色
```python
# 编辑 COLORS 字典
COLORS = {
    'primary': RgbColor(0x1A, 0x5F, 0x7A),  # 修改这里
    ...
}
```

### 添加新页面
```python
# 在适当位置添加
pres.add_content_slide("页面标题", [
    ("要点1", "详细说明1"),
    ("要点2", "详细说明2"),
])
```
